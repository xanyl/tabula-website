Authentication Flow

NextAuth.js v5 with Google + GitHub + email magic link providers

Sign-In Flow

User visits /auth/signin
1 User clicks provider button → NextAuth redirects to OAuth provider
or
1 User enters email → magic link sent via Resend → click link → verified
2 Callback verifies token, checks DB for existing user → upserts user record in `User` table
3 JWT session created → cookie set → redirect to /dashboard

Middleware Protection

Route Protection Matrix
Route No Session Has Session
/, /services, /about, /blog, /case-studies, /contact Allowed Allowed
/auth/signin, /auth/signup Allowed Redirect to /dashboard
/dashboard/* Redirect to /auth/signin Allowed

Session Availability

Server Components
const session = await auth() — available everywhere, no provider needed
Client Components
<SessionProvider> wraps the dashboard layout only. Marketing pages don't need it.
API Routes
auth() directly in route handlers for webhook protection or external API callers

Auth-Aware Navbar

No dedicated auth pages needed in marketing layout
// Navbar (server component) — reads session, renders the right CTA
import { auth } from "@/lib/auth";

export async function Navbar() {
  const session = await auth();

  return (
    <nav>
      ...links...
      {session
        ? <Link href="/dashboard">Dashboard</Link>
        : <Link href="/auth/signin">Sign in</Link>}
    </nav>
  );
}
