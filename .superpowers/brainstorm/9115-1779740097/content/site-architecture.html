Site Architecture & Routing

Next.js App Router with route groups for layout separation

Route Map

URL Structure — Next.js Route Groups
/                          → Homepage (Hero, Services, Process, CTA)
/services                  → Services detail page (expanded from homepage cards)
/about                     → About Tabula (team, mission)
/blog                      → Blog listing with search/filter
/blog/[slug]               → Individual blog post
/case-studies              → Case study listing
/case-studies/[slug]       → Individual case study
/contact                   → Contact form (functional, sends to DB + email)

/auth/signin               → NextAuth.js sign-in page
/auth/signup               → Registration page
/auth/verify               → Email verification

/dashboard                  → Protected: overview, project status
/dashboard/profile          → Protected: account settings
/dashboard/inquiries        → Protected: contact form history
/dashboard/projects         → Protected: project tracking

Middleware Boundary

next.config.js middleware matcher
export const config = {
  matcher: ["/dashboard/:path*", "/auth/:path*"]
}

// Middleware logic:
// /dashboard/*  → check session token, redirect to /auth/signin if absent
// /auth/*       → check session token, redirect to /dashboard if present
// everything else → public
